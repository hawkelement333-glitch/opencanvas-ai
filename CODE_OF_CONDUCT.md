# Code of conduct

## Our standard

OpenCanvas AI contributors and maintainers are expected to create a professional, respectful, and inclusive environment. Productive participation includes:

- discussing ideas and code without attacking people;
- giving specific, actionable, technically grounded feedback;
- respecting differing experience, identity, background, and viewpoint;
- acknowledging mistakes and repairing harm;
- protecting private information, credentials, and security reports;
- crediting work and honoring licenses and ownership.

Unacceptable behavior includes harassment, discrimination, threats, sexualized attention, sustained disruption, deliberate misrepresentation, doxxing, publishing private information, or using project spaces to facilitate abuse.

## Scope

This policy applies in repository discussions, issue trackers, reviews, project chat used for project work, events, demos, and public representation of the project.

## Enforcement

Report conduct concerns privately to the project owner through the approved contact listed in repository metadata. A public reporting address is **TBD and must be added before public release**. Do not include sensitive personal information in a public issue.

Maintainers may edit or remove contributions, restrict participation, or ban a participant when behavior violates this policy. Responses should be proportionate, consistent, and protect reporter confidentiality as far as reasonably possible.

## Good-faith process

Reports should be reviewed promptly and without retaliation. Where safe and appropriate, the reviewer should document the behavior, affected project space, decision, and action. Anyone with a conflict of interest should recuse themselves.

This project-specific policy is original text and is not a claim of affiliation with another community or standards organization.
